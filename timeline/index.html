10互联网产品的用户体验设计（UED）

10.1UED

  • User Experience Design
  • UED是针对用户心灵、眼睛、耳朵、触感的设计
  • 用户体验就是用户在使用一款互联网产品的整体感受度

10.2UED相关概念

  • UE：User Experience
  • UCD：User Centered Design
    • 以用户为中心的设计
    • 强调以用户体验优先的产品设计理念
  • HCI：Human Computer Interaction 人机交互
  • GUI：Graphical User Interface 图形化用户界面
  • WR：Web Rebuild 网站重构

10.3互联网产品的UED流程

  • 产品策划
    • 竞争对手分析
    • 用户研究
  • 产品交互设计
    • 站点信息架构图
    • 用户使用流程图
    • 产品文案规范
    • 原型设计
  • 产品视觉设计
    • 视觉设计
    • 产品还原性评审
    • 产品视觉架构图
    • 视觉规范定制
  • 产品页面重构
    • 视觉设计对重构出来的页面进行确认是否符合当初设计需求
  • 产品研发
    • UE再优化，测试后回炉加工（限于细节上的优化）
  • 产品测试
    • 是否符合视觉规范
    • 是否符合交互需求
    • 联合调试
  • 产品发布
    • 总结
    • 持续跟踪
    • 性能优化

10.4UED团队的常见结构

  • 产品经理
    • 输出
      • 用研文档
      • 竞品分析文档
      • 产品需求与需求分级文档
    • 配合工作
      • 协助UED相关人员对产品的调性进行把控
      • 协助UED相关人员进行测试与组织管理
  • 交互设计
    • 输出
      • 完整产品原型图含交互设计
      • 用户使用流程
      • 产品信息结构图
    • 配合工作
      • 交互测试，测试交互还原度
    • 工作职责
      • 为产品设计具体的行为、界面设计，组织良好的产品信息结构，提高产品的可用性与易用性
  • 视觉设计
    • 输出
      • 产品界面的视觉设计
      • 竞品风格调研
      • 设计规范
      • 概念设计说明
      • 产品流程中界面还原度全程评估审核
    • 配合工作
      • 视觉测试，测试视觉还原度
    • 职责
      • 呈现，美观度，体验，品牌形象设计
  • 网页重构
    • 输出
      • 静态网页
      • 静态页面规范说明（CSS，复用，性能，兼容性等）
    • 配合工作
      • 页面测试工作，测试页面还原度
      • 兼容性测试
      • 页面性能优化
    • 职责
      • 呈现，美观度，体验，品牌形象设计
  • 项目管理

10.5互联网产品的用户体验分类

  • 感官体验：呈现给用户视听上面的体验，强调舒适性，一般包括但不限于：
    • 设计风格
    • 网站LOGO
    • 页面速度
    • 页面布局
    • 页面色彩
    • 动画效果
    • 页面导航
    • 图片展示
    • 图标使用
    • 广告位
    • 背景音乐
    • 等等
  • 交互体验：呈现给用户操作上的体验，强调易用性和可用性，包括但不限于：
    • 会员申请
    • 会员注册
    • 表单填写
    • 表单提交
    • 按钮设置
    • 点击提示
    • 错误提示
    • 在线问答
    • 意见反馈
    • 在线调查
    • 在线搜索
    • 页面刷新
    • 资料安全
    • 显示路径
    • 等等
  • 浏览体验：呈现给用户浏览上的体验，强调吸引性，一般包括但不限于：
    • 栏目的命名
    • 栏目的层级
    • 内容的分类
    • 内容的丰富性
    • 内容的原创性
    • 信息的更新频率
    • 信息的编写方式
    • 精彩内容的推荐
    • 相关内容的推荐
    • 收藏夹的设置
    • 栏目的订阅
    • 信息的搜索
    • 文字排列
    • 文字字体
    • 页面底色
    • 页面的长度
    • 分页浏览
  • 情感体验：呈现给用户心理上的体验，强调友好性以及与用户关系的维系，一般包括但不限于：
    • 友好提示
    • 会员交流
    • 售后反馈
    • 会员优惠
    • 会员推荐
    • 鼓励用户参与
    • 会员活动
    • 专家答疑
    • 邮件/短信问候
    • 网站地图
  • 价值体验：
    • 基础价值
    • 超于基础价值的期望价值
    • 附加价值
  • 信任体验：呈现给用户的信任体验，强调可靠性，一般包括但不限于：
    • 搜索引擎
    • 公司介绍
    • 投资者关系
    • 服务保障
    • 文章来源
    • 文章编辑作者
    • 联系方式
    • 服务热线
    • 有效的投诉途径
    • 安全及隐私条款
    • 法律声明
    • 网站备案
    • 相关链接
    • 帮助中心

12互联网产品的交互设计

12.1交互设计之父比尔.莫格里奇

12.2 交互设计：是用户体验的重要组成部分

  • 互联网交互设计：用户使用产品过程中行为输入产品再到产品输出信息的全过程。

12.3易混淆的概念

  • 产品设计：关注的是功能、结构、人因、形态、色彩，环境等设计要素以及采用的技术、方法和功能的实现手段是间接影响产品最终用户的设计。
  • 以用户为中心的设计：强调用户优先的设计模式
  • 用户体验：努力为用户创造良好的感受
  • 交互设计：注重产品和使用者行为上的交互以及交互的过程，强调的是设计理念和方法。侧重的是交互双方信息表达方式和情感，是直接影响产品最终用户的设计。
  • 人机交互：人与计算机的交互，交互的对象特指计算机，是交互设计里面的一种。
  • 界面设计：主要指人和机器交互界面的设计
  • 可用性：评测用户体验的标准

12.4基础的交互方式

12.5如何更好的学习交互设计

  • 对PM要求
    • 实战经验
    • 知识面广
  • 练习方式：
    • 找到一些常见的任务，例如注册，登陆，发布微博等
    • 仔细分析用户的特征后，明确目标以后，自己开始手绘交互的流程
    • 反复修改，直到你认为已经达到了最简单，最好体验
    • 对比你的设计与真正现有产品

12.6常用交互设计的原则

  • 可视性：功能可视性越好，越方便用户发现和了解使用方法
  • 反馈：返回与活动相关的信息，以便用户能够继续下一步操作
  • 限制：在特定时刻显示用户操作，以防误操作
  • 映射：准确表达控制及其效果之间的关系
  • 一致性：保证同一系统的同一功能的表现及操作一致
  • 启发性：充分准确的操作提示

12.7交互设计准则

  • 有关爱
    • 不伤害
    • 改善人的状况
  • 有意图的帮助用户实现他们的目标和渴望
  • 注重实效
  • 优雅
    • 最简单的完整方案
    • 内部一致性
    • 合适的容纳和情感
